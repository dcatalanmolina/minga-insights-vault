This repo shows how to discover, write, and organize user insights using Obsidian and one of its community plugins, [Quadro](https://github.com/chrisgrieser/obsidian-quadro).

Quadro allows you to code text from interviews, meeting notes, and other sources. You can then organize the coded data into insights that guide product development.

# How to analyze and organize insights

- Use the `Projects` folder to share an overview of the projects associated with the data in the repo.
- Use the `Data` folder to add new documents with interview transcripts, meeting notes, and other sources of qualitative data. Within each document, highlight paragraphs or sentences to code into different themes. 
- Everything you code will be automatically organized in the `Codes` folder. Quadro does this for you and you shouldn't edit documents in the `Codes` folder.
- Create a new document under `Analysis` to take notes. Your notes will depend on the question you are answering and the themes and categories you notice inside `Codes`.
- Synthesize your analyses by writing insights in a format that makes product development easier. For example, use the following sentence structure to title an insight and to organize themes under it:

> _As a (role) I need a way to (achieve goal) in order to (fulfill underlying motivation)_

- Create a new canvas in `Global Notes` to organize insights and to facilitate discussion with others.
